# LINUX DO 默认树形评论区1.7.0

Tampermonkey 用户脚本，用于在访问 LINUX DO 的 Discourse 话题页时，默认打开树形评论区，并补齐树形视图下常用的导航和搜索能力。

## 功能

- 自动把普通话题链接从 `/t/...` 改写为 `/n/...`，进入 Discourse 树形评论区。
- 不依赖论坛页面里的“以嵌套方式查看”按钮；只要站点仍支持 `/n/...` 树形路由，脚本会直接通过 URL 改写进入树形视图。
- 保留用户主动进入平面视图的意图，包括“View as flat / 以平面方式查看 / 平面图查看”等链接。
- 访问树形话题页时恢复正确的浏览器标题，避免标题只显示站点名。
- 在话题页顶部搜索入口旁新增 `L` 话题工具按钮，嵌套视图和平面视图都会显示；点击后可选择“本话题搜索”或“以平面方式查看”。
- 支持通过 Discourse 搜索接口查找当前话题所有回复，包括尚未加载到页面里的回复。
- 搜索结果同时提供“嵌套查看”和“平面查看”，并可定位到对应楼层。

## 限制

以下限制同时作为基础行为边界，后续修复 bug 或新增功能时不应破坏。

- 不适合树形视图的话题必须保持原始平面话题地址。脚本会读取 `/t/{topicId}.json` 的话题数据，私信话题（`archetype: private_message`）、Banner 话题（`archetype: banner`）、Post Voting / 问答投票话题（`is_post_voting` 或 `subtype: question_answer`）、普通投票帖（首帖或帖子数据里包含 `polls` / `polls_votes` / `has_polls` 等投票字段）不会改写到 `/n/...`。
- Discourse 通用 `topic` 占位 slug 不再被视为不支持树形视图。已验证 `/n/topic/{topicId}` 可正常打开时，`/t/topic/{topicId}` 和相关站内链接应继续自动切换到嵌套模式。
- 从提醒列表、通知菜单、用户动态等入口进入帖子时，也应以嵌套方式查看；如果入口带有楼层信息，应在树形话题中定位到对应楼层。
- 在话题列表中点击进入帖子时，应显示完整树形话题，而不是只打开单一话题串。列表入口中的 `/t/.../{postNumber}` 或 `data-post-number` 只用于 Discourse 的列表定位语义，不能被当成搜索结果式的精确楼层跳转。
- 搜索结果中的“嵌套查看”是精确楼层跳转，应保留楼层号并滚动到匹配回复；“平面查看”应临时尊重用户进入平面视图的意图。
- 脚本生成的平面视图入口会在 URL 上附带 `linuxdo_flat=1` 标记，用于穿过 Discourse SPA 路由和页面重载后继续保持平面视图；标记被消费后会写入平面视图短期锁，并从地址栏移除，避免后续路由更新又自动切回嵌套视图。
- 用户手动点击“View as flat / 以平面方式查看 / 平面图查看”等平面视图入口后，同一话题的后续 SPA 路由更新不应自动切回嵌套视图；用户明确打开嵌套话题地址后，可以恢复默认树形行为。
- 在一个树形话题中跳转到另一个话题时，应避免复用旧话题的树形视图状态，防止新话题内容混入旧话题的回复树。

## 安装

1. 安装 Tampermonkey 或兼容的用户脚本管理器。
2. 新建用户脚本。
3. 复制 `LinuxdoComment.js` 的全部内容并保存。
4. 打开 `https://linux.do/` 的话题列表或话题页验证效果。

## 验证

```bash
node --test LinuxdoComment.test.js
node --check LinuxdoComment.js
```
